Subject: [PATCH v5 00/11] xen: Add NUMA-aware claim sets for domains

Introduction:
-------------

Dear Reviewers,

Thank you for your feedback on the previous series. This update builds
on that feedback to support installing multi-node claim sets atomically.

The patch structure has been revamped, and the implementation has been
improved based on your suggestions.

Patch Summary:
--------------

This series begins with comprehensive design documentation to assist your
review. I recommend using the rendered documentation as a reference, as
it addresses many potential questions. You can view it on ReadTheDocs or
render it locally using the instructions below.

The series contains the following patches:

1. docs/design:      Add design documentation for NUMA-aware claim sets
2. xen/page_alloc:   Extract releasing global claims into a function
3. xen/page_alloc:   Recall global claims if needed when offlining pages
4. xen/page_alloc:   Fix to add unclaimed pages when allocating with claims
5. xen/page_alloc:   Optimize getting per-NUMA-node free page counts
6. xen/page_alloc:   Split d->outstanding_pages: global and per-node claims
7. xen, tools:       Add XEN_DOMCTL_claim_memory with legacy claims
8. xen/page_alloc:   Add the implementation of NUMA-node-specific claims
9. tools/ocaml/libs: Add the OCaml binding for xc_domain_claim_memory()

Review:
-------

Given the significant changes in this series to support multi-node claim sets,
I recommend reviewing it as a whole rather than incrementally.

The design documentation is available here to assist your review:
https://bernhardk-xen-review.readthedocs.io/node-claims-v5/designs/claims/

I recommend using the rendered design documentation as a reference,
as it should address most potential questions.

In addition to rendering it on ReadTheDocs, you can also render it locally:

./configure && python -m venv .sphinx-docs && source .sphinx-docs/bin/activate
pip install sphinx-rtd-theme sphinxcontrib-mermaid && make -C docs sphinx-html
