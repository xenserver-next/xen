Subject: [PATCH v5 00/11] xen: Add NUMA-aware memory claims for domains

This is v5 of this series.
Feedback on v4 resulted in several follow-up changes to the core of this
series. These are small, non-breaking changes. A full list of v5 changes is
included below.

This series introduces NUMA-aware memory claims and a hypercall API for them.

Patches (one patch was split, one removed, and one OCaml test added):

 1. xen/page_alloc:  Extract consuming & releasing claims into functions
 2. xen/page_alloc:  Add node_avail_pages[], use it in avail_node_heap_pages()
 3. xen/page_alloc:  Make total_avail_pages and outstanding_claims unsigned long
 4. xen/page_alloc:  Add the implementation of NUMA-node-specific claims
 5. xen/domain:      Add the XEN_DOMCTL_claim_memory hypercall handler
 6. xsm/flask:       Add a Flask security policy for the new hypercall
 7. tools/libs/ctrl:     Add the libxenctrl API xc_domain_claim_memory()
 8. tools/ocaml/libs/xc: Add the OCaml binding for xc_domain_claim_memory()
 9. tools/tests:         Add testing per-node claims and claims protection
10. tools/tests/ocaml:   Add OCaml test suite infrastructure with a claims test
11. docs/guest-guide:    Add comprehensive claims and claims API documentation

The patches are ordered from low-level preparation to implementation to tests
and documentation. They can be applied over time, but for commit-history
readability it is best (not required) to keep them in order.

The updated guest-guide is deployed here for reviewing the created output:
https://bernhardk-xen-review.readthedocs.io/v4.22-claims.v5/guest-guide/
